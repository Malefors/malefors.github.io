Automatic search with an optimiser

StochSD has an Optimisation tool which is found under the menu: Tools/Optim, which it is now time to use. The Tools menu looks like the Figure below.



Figure. The StochSD Tools Menu.

Click the Optim button in the Tools menu! This opens the Optimisation Tool shown below.



Figure. The optimisation tool named “Optim”in StochSD.

We have created V as our “Objective function” which we now want to Minimise. In the “Param. Name” box we now write a name K1, one of the parameters we want to change. In the “Start” box, fill in your best guess of the value of K1 and in the “Init Step” box, you fill in a value approximately 1/10 of your guessed value of K1. The reason why we put in 1/10 of K1 is because we do not want the optimiser to search for a solution with negative K1 values and there is a risk of K1 quickly becoming negative if we start with a to big initial change in K1. Now, click the “add”-buttton and repead the process and add a similar row for K2 since these are the parameters we want to estimate.

Check that you have created V as your “Objective function” which we now want to Minimise. In the “Param. Name” box check that you have two rows in which you have written “K1”and “K2”. The optimiser also needs information on how precise, or near to the global optimum, the solution needs to be, in order to meet a stop criteria for its search. In the “Required Accuracy” box you fill in a small value (for instance 1E-4) which corresponds to the uncertainty you can accept in the determination of V. A better precision requires longer time.

When the optimization dialogue window is completed, Click the “Optimise” button.

When the optimiser has finished, use for example Window Button shift+s to harvest the results from the optimiser.

Run the model with the new values of K1 and K2 and check that the model behaves similarly as the “reality”. If you run into trouble ask the teacher for help!

When your model behaves as “reality”, use the estimated values K1 and K2 in the further development of the model in the next exercise.
